Required values:

$siteId
$webId
$listId
$webUrl
$webTitle


Steps:

1. Open the SharePoint document library in Microsoft Edge or Google Chrome.

2. Press F12 to open Developer Tools.

3. Go to the Network tab.

4. Make sure network recording is enabled.

5. Click the Sync button in SharePoint.

6. When prompted, allow the browser to open Microsoft OneDrive.

7. In the Network tab, search for:

   odopen

8. Find the request that starts with:

   odopen://sync/?

9. Copy the full odopen:// URL.

10. Pull the needed values from that URL and place them into the script.  The script ODopen_extract.ps1 will extract these values for you

Example placeholder values:

$siteId = "{SITE-ID-GUID}"
$webId  = "{WEB-ID-GUID}"
$listId = "{LIST-ID-GUID}"

$webUrl = "https://yourtenant.sharepoint.com/sites/YourSiteName"
$webTitle = "Your Site Title"

Important:

The values in the odopen:// URL are URL-encoded.

For example:

%2D = -

This means a value such as:

12345678%2D1234%2D1234%2D1234%2D123456789012

Should be converted to:

12345678-1234-1234-1234-123456789012

You can either:
- Leave the values URL-encoded if using the full odopen URL directly
- Or replace the encoded values with normal characters when manually filling variables in the PowerShell script

Common URL-encoded characters:

%2D = -
%20 = space
%3A = :
%2F = /

Example:

Encoded:
Project%20Files

Decoded:
Project Files
